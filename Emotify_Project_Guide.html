1. What is Emotify? (Project Overview)

Emotify is a web-based music recommendation system that uses your webcam to detect your facial emotion in real-time and then recommends a curated playlist of songs that match your current mood.

The Problem It Solves

Manually searching for songs that match your mood is tedious. Emotify automates this — it reads your face like a friend and plays songs to match how you're feeling.

How It Works (In Simple Terms)

  1. You open the website and click "Try Emotify"
  2. Your webcam opens and detects your face
  3. A trained AI model reads your facial expression and classifies your emotion (Happy, Sad, Angry, etc.)
  4. The emotion is mapped to a mood category (Happy, Sad, Energetic, Calm)
  5. The app picks 10 songs (5 Bollywood + 5 Hollywood) matching that mood from a dataset
  6. Songs are displayed with embedded Spotify players so you can listen instantly

  • Opens the webcam using OpenCV
  • Uses CAP_DSHOW (DirectShow) on Windows for better camera compatibility

Step 2: Detect Faces (Loop for ~5 seconds)

face_classifier = cv2.CascadeClassifier(str(CASCADE_PATH))
gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
faces = face_classifier.detectMultiScale(gray, scaleFactor=1.1, minNeighbors=4)
  • Converts each video frame to grayscale (face detection works better without color)
  • Uses the Haar Cascade classifier to find face rectangles in the frame
  • scaleFactor=1.1 — searches for faces at 10% scale increments
  • minNeighbors=4 — requires 4 neighboring detections to confirm a face (reduces false positives)

Step 3: Classify Emotion

roi_gray = cv2.resize(roi_gray, (48, 48))           # Resize to 48×48
roi = roi_gray.astype("float32") / 255.0              # Normalize to 0-1
roi = np.expand_dims(roi, axis=(0, -1))                # Shape: (1, 48, 48, 1)
prediction = classifier.predict(roi, verbose=0)[0]     # CNN inference
label = EMOTION_LABELS[int(prediction.argmax())]        # Get emotion name
  • Extracts the face region from the frame
  • Resizes it to 48×48 pixels (matching the CNN's expected input size)
  • Normalizes pixel values from [0-255] to [0.0-1.0] (helps the model perform better)
  • Adds batch and channel dimensions → shape becomes (1, 48, 48, 1) where 1=batch, 48×48=spatial, 1=grayscale channel
  • Feeds it to the CNN model which outputs 7 probabilities
  • Takes the highest probability as the predicted emotion

Step 4: Vote & Pick Dominant Emotion

counts = {}
counts[label] = counts.get(label, 0) + 1  # Count each detection
emotion = max(counts.items(), key=lambda item: item[1])[0]  # Most frequent
  • Over ~5 seconds, multiple frames are analyzed
  • Each frame's emotion detection is counted as a vote
  • The emotion with the most votes is chosen as the final result
  • This voting approach makes the result more reliable than a single-frame prediction

Step 5: Get Song Recommendations

mood = normalize_mood(emotion)       # e.g., "Happy" → "Happy"
recommendation = build_recommendation(mood=mood, limit=10)
  • Maps the detected emotion to a mood category
  • Fetches 10 matching songs from the CSV dataset

Step 6: Display Results
Returns a styled HTML page showing:

  • The detected emotion with a mood badge
  • 5 Bollywood song cards with embedded Spotify players
  • 5 Hollywood song cards with embedded Spotify players

4.4 The CNN Model — Deep Learning (model.h5)

What is a CNN?

A Convolutional Neural Network is a type of deep learning model specifically designed for processing images. It works by:

  1. Learning to detect simple features first (edges, corners)
  2. Combining them into complex features (eyes, nose, mouth shapes)
  3. Using those to make a classification (which emotion?)

Model Architecture (1,328,167 total parameters)

Input: 48×48×1 grayscale face image
    ↓
┌─────────────────────────────────────────────────────┐
│  BLOCK 1: Conv2D(32) → ReLU → BatchNorm → Conv2D(32) → ReLU → BatchNorm  │
│  → MaxPool(2×2) → Dropout(25%)                                            │
│  Output: 24×24×32                                                          │
├─────────────────────────────────────────────────────┤
│  BLOCK 2: Conv2D(64) → ReLU → BatchNorm → Conv2D(64) → ReLU → BatchNorm  │
│  → MaxPool(2×2) → Dropout(25%)                                            │
│  Output: 12×12×64                                                          │
├─────────────────────────────────────────────────────┤
│  BLOCK 3: Conv2D(128) → ReLU → BatchNorm → Conv2D(128) → ReLU → BatchNorm│
│  → MaxPool(2×2) → Dropout(25%)                                            │
│  Output: 6×6×128                                                           │
├─────────────────────────────────────────────────────┤
│  BLOCK 4: Conv2D(256) → ReLU → BatchNorm → Conv2D(256) → ReLU → BatchNorm│
│  → MaxPool(2×2) → Dropout(25%)                                            │
│  Output: 3×3×256                                                           │
├─────────────────────────────────────────────────────┤
│  Flatten → 2304 neurons                                                    │
│  Dense(64) → ReLU → BatchNorm → Dropout                                   │
│  Dense(64) → ReLU → BatchNorm → Dropout                                   │
│  Dense(7) → Softmax → Output probabilities for 7 emotions                 │
└─────────────────────────────────────────────────────┘

Layer-by-Layer Explanation

Layer Type What It Does Why It's Used
Conv2D Slides small filters (3×3) across the image to detect features like edges, textures, and patterns Core building block of CNNs — learns spatial patterns automatically
ReLU Activation Replaces negative values with zero: f(x) = max(0, x) Introduces non-linearity, allowing the model to learn complex patterns. Computationally cheap.
Batch Normalization Normalizes the output of each layer to have mean ≈ 0 and std ≈ 1 Speeds up training, stabilizes learning, reduces sensitivity to weight initialization
MaxPooling2D (2×2) Takes the maximum value in each 2×2 window, halving spatial dimensions Reduces computation, makes features translation-invariant (face can be slightly shifted)
Dropout (25%) Randomly "turns off" 25% of neurons during training Prevents overfitting — forces the model to not rely on any single neuron
Flatten Converts 3D feature maps (3×3×256) into a 1D vector (2304) Required to connect convolutional layers to fully connected (Dense) layers
Dense (64) Fully connected layer — every neuron connects to every neuron in the previous layer Combines all learned features for final classification
Dense (7) + Softmax Output layer with 7 neurons, one per emotion. Softmax converts to probabilities (sum = 1.0) Gives a probability distribution over all 7 emotions

Model Stats

  • Total parameters: 1,328,167 (~1.3 million)
  • Trainable parameters: 1,325,991
  • Model file size: 15.4 MB
  • Accuracy: ~72% on the FER-2013 dataset
  • Input: 48×48 pixel grayscale face images
  • Output: 7 emotion probabilities

Training Dataset — FER-2013

The model was trained on the FER-2013 (Facial Expression Recognition 2013) dataset:

  • 35,887 grayscale images, each 48×48 pixels
  • 7 categories: Angry, Disgust, Fear, Happy, Neutral, Sad, Surprise
  • Collected from Google Image Search
  • Standard benchmark in the facial emotion recognition field

4.6 K-Means Clustering (How Songs Were Categorized by Mood)

The clustering2.ipynb notebook shows how songs were assigned to mood categories.

What is K-Means?

An unsupervised machine learning algorithm that groups data points into K clusters based on similarity. It works by:

  1. Randomly placing K centroids
  2. Assigning each data point to the nearest centroid
  3. Moving centroids to the mean of their assigned points
  4. Repeating steps 2-3 until convergence

How It Was Applied

from sklearn.cluster import KMeans
from sklearn.preprocessing import MinMaxScaler

# Audio features used for clustering
features = ['danceability', 'energy', 'valence', 'loudness']

# Normalize to 0-1 range
X = MinMaxScaler().fit_transform(df[features])

# Cluster into 4 groups (matching our 4 moods)
kmeans = KMeans(init="k-means++", n_clusters=4, random_state=15).fit(X)
df['kmeans'] = kmeans.labels_

Audio features used:

Feature Range What It Measures
Danceability 0.0–1.0 How suitable a track is for dancing
Energy 0.0–1.0 Intensity and activity level
Valence 0.0–1.0 Musical positiveness (happy vs sad sound)
Loudness -60 to 0 dB Overall volume

Cluster interpretation:

  • High valence + high energy + high danceability → Happy
  • Low valence + low energy → Sad
  • High energy + low valence → Energetic
  • Low energy + medium valence → Calm

8. Potential Viva Questions & Answers

Basic Questions

Q1: What is the main objective of Emotify?

To build a music recommendation system that uses facial emotion detection through a webcam to automatically suggest songs matching the user's current mood.

Q2: What technologies does Emotify use?

Frontend: React.js with Bootstrap. Backend: Python Flask, TensorFlow/Keras for deep learning, OpenCV for face detection, Pandas for data handling, Spotipy for Spotify API integration.

Q3: How many emotions can the system detect?

7 emotions: Angry, Disgust, Fear, Happy, Neutral, Sad, Surprise.

Q4: How many moods are songs categorized into?

4 moods: Happy, Sad, Energetic, and Calm. The 7 emotions are mapped down to these 4 mood categories.

Q5: What dataset was the emotion model trained on?

FER-2013 (Facial Expression Recognition 2013) — contains 35,887 grayscale face images at 48×48 pixels across 7 emotion categories.

Intermediate Questions

Q6: Why did you use a CNN instead of a traditional ML model?

CNNs are specifically designed for image data. They automatically learn spatial hierarchies of features (edges → textures → face parts → expressions) through convolutional filters, whereas traditional ML models would need manual feature extraction (which is hard for faces).

Q7: What is Haar Cascade and why did you use it?

Haar Cascade is a pre-trained classifier in OpenCV that detects faces using edge and line features in a cascade of stages. We chose it because it's fast (real-time), lightweight (no GPU needed), and reliable for frontal face detection.

Q8: Why do you normalize pixel values to 0-1?

Neural networks train faster and more stably when inputs are in a small, consistent range. Raw pixel values (0-255) can cause large gradient updates. Dividing by 255 maps everything to [0, 1].

Q9: What is Batch Normalization and why is it used?

Batch Normalization normalizes the outputs of each layer to have zero mean and unit variance. This prevents "internal covariate shift" (layer input distributions changing during training), allowing higher learning rates and faster convergence.

Q10: What is Dropout and why is it important?

Dropout randomly deactivates a percentage of neurons during training. This prevents overfitting by ensuring no single neuron becomes overly important. It acts as a regularization technique, making the model generalize better to unseen faces.

Q11: How does K-Means decide which mood a song belongs to?

K-Means groups songs based on audio features (danceability, energy, valence, loudness). Songs that are "close" in this feature space get assigned to the same cluster. Each cluster naturally corresponds to a mood — e.g., high-energy high-valence songs cluster together → Happy.

Q12: Why do you use a voting mechanism over 5 seconds?

A single frame can have noise, bad lighting, or momentary expression changes. By counting emotion detections over multiple frames and choosing the most frequent one, we get a much more reliable prediction.

Advanced Questions

Q13: What is the accuracy of the model?

Approximately 72% on the FER-2013 test set. This is competitive — FER-2013 is a challenging dataset because many images are ambiguous even to humans (inter-rater agreement is ~65%).

Q14: How many parameters does the CNN have?

1,328,167 total parameters (~1.3 million), of which 1,325,991 are trainable. The model file is 15.4 MB.

Q15: Can you explain the architecture in one sentence?

Four convolutional blocks (32→64→128→256 filters) each with double Conv2D + BatchNorm + MaxPool + Dropout, followed by a Flatten layer and two Dense(64) layers, outputting 7 softmax probabilities.

Q16: What is Softmax?

Softmax converts the raw outputs (logits) of the final layer into probabilities that sum to 1. If the logit for "Happy" is highest, softmax will give it the highest probability (e.g., 0.85), and the rest will share the remaining 0.15.

Q17: What is MaxPooling and why 2×2?

MaxPooling takes the maximum value in a 2×2 window, reducing spatial dimensions by half. This gives translation invariance (the model doesn't care if the eye is shifted by 1 pixel), reduces computation, and helps prevent overfitting.

Q18: What is OAuth 2.0?

OAuth 2.0 is an authorization framework that lets a third-party app (Emotify) access a user's Spotify account without seeing their password. The user grants permission through Spotify's login page, and Emotify receives a temporary access token.

Q19: Why Flask instead of Django?

Flask is lightweight and minimal — perfect for a focused API server with just 3-4 routes. Django would be overkill (includes ORM, admin panel, template engine we don't need) and add unnecessary complexity.

Q20: What are the limitations of Emotify?

(1) Requires good lighting for face detection, (2) Works best with frontal faces only, (3) The 72% accuracy means some emotions may be misclassified, (4) Limited Bollywood song database (only 20 songs), (5) Requires a Spotify account for playlist creation.

Q21: How would you improve Emotify?

(1) Use a deeper model like VGG-16 or ResNet for better accuracy, (2) Add more songs especially Bollywood, (3) Use content-based filtering with user history, (4) Support multiple faces simultaneously, (5) Add a mobile app version, (6) Use transfer learning from pre-trained face recognition models.
